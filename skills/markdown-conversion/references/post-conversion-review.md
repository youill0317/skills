# Post-Conversion Review

Load this reference after initial conversion to decide the next step.

## Review Checklist

1. The document begins and ends where the source begins and ends.
2. Major sections and headings are present.
3. Lists still read like lists rather than collapsed paragraphs.
4. Tables remain understandable, even if simplified.
5. Links, images, code blocks, and obvious identifiers were not dropped.
6. No large chunk of content appears truncated or silently omitted.

## Route to Markdown Format Normalization When

- Content appears complete.
- Structure is understandable.
- Problems are mostly spacing, heading depth, list markers, table formatting, or frontmatter consistency.

## Route to Markdown Source-Aligned Repair When

- Content appears missing or mistranscribed.
- OCR or extraction noise remains visible.
- Paragraphs, lists, or tables collapsed in ways that change meaning.
- The user explicitly wants the Markdown checked against the original source.
