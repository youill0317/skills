# Frontmatter and Tags

Load this reference when the Markdown document has YAML frontmatter or should maintain consistent tags.

## Frontmatter Rules

1. Keep frontmatter at the top of the file.
2. Preserve existing keys unless the user requests removal or the key is clearly malformed duplication.
3. Normalize YAML list formatting for `tags`.
4. Do not invent metadata fields beyond what the task requires.

## Tag Rules

1. Preserve existing valid tags.
2. Normalize separator and spacing style consistently.
3. Add or rename tags only when the task explicitly asks for it or a fixed project convention is already present.
