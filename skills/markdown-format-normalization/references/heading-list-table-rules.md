# Heading, List, and Table Rules

Load this reference when major Markdown block structure is inconsistent.

## Headings

1. Use exactly one H1 only when the document clearly has a single title.
2. Keep heading depth sequential where possible.
3. Do not promote or demote headings arbitrarily; follow the visible document hierarchy.

## Lists

1. Use `-` for unordered lists unless the user requests another style.
2. Use `1.`, `2.`, `3.` format for ordered lists.
3. Indent nested lists consistently by four spaces.
4. Preserve checklist syntax if the source uses task lists.

## Tables

1. Normalize header separators and column counts.
2. Preserve cell text exactly unless spacing damage must be repaired.
3. If the source structure is not representable as a Markdown table, convert it to a list rather than forcing a broken table.

## Callouts

Represent note-like boxes as blockquotes with a bold label when no other house style is specified.
